# Setup Instructions
This guide provides detailed, step-by-step instructions for setting up OnionC2, a command and control (C2) tool designed to manage remote systems over the Tor network. The process involves configuring the C2 server, setting up a Tor onion service, and compiling and running the agent. These instructions assume you have basic familiarity with command-line tools and the required programming environments.

## Prerequisites
Before starting, ensure the following software is installed on your system:
- Go compiler (for api)
- Rust compiler (for agent)
- Tor (for onion service setup)
- XENA (for the user interface)

You will also need:
- A Unix-like environment (Linux recommended)
- Basic command-line proficiency

## Step 1: Clone the Repository
Begin by downloading the OnionC2 project files from GitHub:
- git clone https://github.com/zarkones/OnionC2.git
- cd OnionC2

This creates a local copy of the repository on your system.

## Step 2: Set Up the C2 Server
The C2 server handles communication between the agents and the user interface. To configure it:

Navigate to the api directory and launch the C2 server with a custom API key (must be longer than 16 characters):
- cd api
- go run . --api-key=your_secret_api_key_longer_than_16_chars

Replace your_secret_api_key_longer_than_16_chars with a secure, unique key of your choosing.

The server will now start and listen for connections from the agent and the user interface.

## Step 3: Configure the Tor Onion Service
OnionC2 uses a Tor onion service for anonymous communication. After you run the C2 it will generate a torrc file in the "api" directory. Make the Tor setup an onion service based on that config by running the following command inside of the "api" directory:
- tor -f torrc

Retrieve the .onion address generated by Tor by running:
- cat api/onionservice/hostname

You'd use that hostname to configure an agent inside of a file named "agent/src/config.rs" under function named "get_address".

# Step 4: Compile Agent
To compile an agent enter its directory and run the "build.sh" script:
- sh build.sh

If you don't have the bash script you'd be better of manually running the command from inside of that script:
- RUSTFLAGS="-Zlocation-detail=none" RUSTFLAGS="-Zfmt-debug=none" cargo build --release --target x86_64-pc-windows-gnu

# Step 5: Integrate with XENA for the User Interface
OnionC2 uses XENA as its user interface for managing operations. To set it up install XENA following its official setup instructions.

Once you run it go to settings tab and put your_secret_api_key_longer_than_16_chars into "Authentication Token" field. (Alternatively the UI accepts it via AUTH_TOKEN environment variable)

# Additional Notes
Agent is meant to be highly configurable. File "agent/src/config.rs" is well documented via code comments and explains which options does what. Even if you aren't familiar with Rust programming language you'd be able to make the best use of it. If you have any questions reach out to us on Discord or an AI like Grok.
